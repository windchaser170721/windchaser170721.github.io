调研 ​

主线逐渐分成三层：第一层是给 VLA 注入几何先验，第二层是给 VLA 注入显式运动表征，第三层是直接学 3D world model / world action model。SpatialVLA、PointVLA、GeoAware-VLA、FALCON、3D-Mix、R3DP、LaMP、ParticleFormer、PointWorld、DreamZero、Ctrl-World 基本把这三层都占住了。它们共同说明，3D 不再只是 perception enhancement，而是在往 policy interface、motion prior、dynamics model 这三个位置迁移。

3D 不是额外注入，是决策前一种结构化中间层，结构的方式很关键。PointVLA 把 3D 注入点放在 action expert 而不是重训整个 VLA；FALCON 进一步把 spatial tokens 专门送进 Spatial-Enhanced Action Head，刻意避免破坏 vision-language 对齐；3D-Mix 又把这个问题系统化，比较了 9 种 VGGT 融合方式，最后发现最稳的是 semantic-conditioned gated fusion；R3DP 则把3D 很有用但太慢的现实问题显式提出来，用 fast-slow 异步设计把大 3D 模型和实时控制拆开。现在的共识：几何信息很重要，但它最适合在一个受控的接口里发挥作用，而不是粗暴塞满整个 backbone。

几何信息 3D信息 motion 之间是什么关系

如果项目想做大，我最看好的三条主线

第一条主线，是做 geometry-conditioned VLA，重点不是加 3D encoder，是研究几何先验进入 policy 的最佳接口。它既承接了 SpatialVLA、PointVLA、GeoAware-VLA、FALCON、3D-Mix、R3DP 这批工作的主流趋势，也很容易形成你自己的方法学问题：

  • 3D 信息到底应该进 vision backbone、language-action bridge、action head，还是作为 latent control variable；什么时候该稀疏更新，什么时候该逐帧更新；
  • 不同几何表征如 depth、point token、scene flow、camera-aware feature 哪种更适合 manipulation。

GeoAware-VLA 在 unseen viewpoint 上报告了超过 2 倍的零样本泛化增益，3D-Mix 在 OOD SIMPLER 上给出平均 +7.0% 的提升，R3DP 则显示用异步机制可以在提升成功率的同时把推理时间再降 44.8%。

第二条主线，是做 motion-prior VLA，模型不只看懂几何，让它显式拥有一个 3D 运动先验。

  • LaMP 直接把 dense 3D scene flow 当成 latent motion prior，让 Motion Expert 的隐藏状态去条件化 Action Expert，并在 LIBERO、LIBERO-Plus、SimplerEnv-WidowX 上都超过比较基线，在 OOD 扰动上相对最强基线平均再涨 9.7%。
  • SG-VLA 另一种做法是不用显式 scene flow，而是通过多视角、depth、短时历史加辅助解码任务，把目标物相对位姿、grasp affordance、global robot pose 等中间信号一起学出来。

它们共同指向一个问题：与其让 action head 从 2D token 里 自己悟出物体怎么动 ，不如直接给它一个运动相关的 bottleneck。这条线更接近真正的 manipulation intelligence，因为接触、遮挡、形变和时序依赖本质上都是运动建模问题，不只是空间理解问题。

Tip: MPC 是什么，为什么这里总在提它？

PointWorld：将 state 和 action 统一到 shared 3D space 中。

MPC 是 Model Predictive Control，也就是“模型预测控制”。它不是先一次性规划完整条轨迹再完全执行，而是在每个控制时刻都做一个短视野规划：先用当前状态和动力学模型预测未来几步会发生什么，再从一批候选动作序列里选出当前最优的一条，但通常只执行第一步，然后立刻重新观测、重新规划。

所以 MPC 的核心不是“规划一次”，而是边预测、边执行、边修正。这和机器人操作很契合，因为接触误差、视角变化、物体滑动、感知噪声都很常见，闭环重规划通常比开环执行更稳。

放到这里看，PointWorld 这类方法的重要性在于：它们试图提供一个足够快、又足够贴近 3D 物理变化的未来预测模型。只要这个模型能在短时间内稳定预测 “给定动作后场景会怎么变”，MPC 就能直接拿它来滚动评估候选动作，因此 world model 不只是辅助理解，而是可以直接进入决策环。

第三条主线，是做显式 3D world model / world action model。这条线更难，但上限也最高。ParticleFormer 已经把 3D world model 从单材料物体推进到 rigid、deformable、flexible 混合的 multi-object / multi-material 交互，而且可以直接接 MPC；PointWorld 更进一步，把 state 和 action 统一成 shared 3D space 里的 point flow，用约 200 万条轨迹和 500 小时真实与仿真数据训练，在 0.1 秒级推理下直接服务 manipulation MPC；DreamZero 则代表 video-based WAM 路线，它基于预训练 video diffusion 联合预测 future world states 和 actions，在 real robot 上对新任务、新环境泛化相对 SOTA VLA 超过 2 倍，并实现 14B 模型 7Hz 闭环控制。换句话说，这条线的核心命题已经不是“VLA 怎么更聪明”，而是“policy 要不要退居二线，让 world dynamics 成为主角”。

只做 geometry fusion，现在已经很卷了，PointVLA、FALCON、3D-Mix、R3DP 这些工作把 3D 特征怎么接 这件事做得越来越细，再做一个 某种新融合块 容易落到 incremental。如果一上来就做 DreamZero 那种视频 WAM 或 PointWorld 那种大规模 foundation 3D world model，代价又太高，对数据、算力、系统能力要求都很重。中间这条路更适合长期项目：先把 3D motion prior 作为 action 的中间监督或 latent variable，用它稳定提升 VLA 的 OOD manipulation；再逐步把这个 motion prior 扩成短时 horizon 的局部 3D world model，最后接 planner / MPC / reranker。这条路线既能吃到当前趋势，又比较符合 从 idea 到系统 的自然演化。

第一个核心问题，**3D 应该以什么形式进入控制？**可以是 point token，可以是 geometry feature，可以是 scene flow，可以是 point flow，也可以是 video latent。PointWorld 押的是 state-action unified 3D point flow，LaMP 押的是 scene flow latent motion prior，DreamZero 押的是 joint video-action modeling。这说明真正有意思的研究问题已经从 模态融合 变成了 动作条件下的未来世界表示。

第二个核心问题，3D 先验要显式还是隐式。GeoAware-VLA 和 FALCON 更偏隐式先验：从冻结的几何 foundation model 里抽 feature，再交给 policy；ParticleFormer、PointWorld 更偏显式 3D dynamics：直接在点云/点流空间预测未来。前者更容易接现有 VLA，后者更适合讨论 contact dynamics、跨视角一致性、跨 embodiment 迁移。这个决定了后面是更像一个 robotics policy project ，还是更像一个* embodied world model project*。

第三个核心问题，实时性和几何精度怎么平衡。R3DP 专门提出 fast-slow collaboration，就是因为直接把 VGGT 一类大 3D 模型逐帧塞进控制会导致时延太高；PointWorld 则证明如果 world model 设计得对，0.1 秒级推理是可以服务 MPC 的；DreamZero 又用系统优化把 14B video diffusion 压到了 7Hz。这个问题非常现实，也非常研究。因为很多 几何很强 的方法，一旦进闭环，马上就被时延、视角同步、相机标定、深度噪声打回原形。真正能做成长期项目的方案，必须从一开始就把 latency 当一等公民。

如果是我来定一个长期题目，我会怎么命名

我会倾向于把题目落在这种风格上：

“面向具身操作的 3D 运动先验驱动视觉语言动作模型研究”

或者再往 world model 方向靠一点：

“基于显式 3D 运动表征的具身世界动作模型研究”

前者更稳，后者更新也更有上限。前者的关键词是 VLA、3D prior、scene flow、action head、OOD generalization；后者的关键词是 short-horizon world model、point flow、action-conditioned dynamics、MPC / reranking / planning。你后面可以根据资源再决定是以 VLA 为主体，还是以 world model 为主体。这个区分在 2025–2026 的文献里已经很清楚：FALCON、3D-Mix、R3DP 更像前一类，PointWorld、ParticleFormer、DreamZero、Ctrl-World 更像后一类。

我的一个比较直接的 sense

我现在越来越不看好“单纯给 VLA 加 3D token”作为长期主线，因为这条线很快会同质化。真正更值得做大的，是把 3D 从 perception signal 升级成 decision variable。也就是说，不是“模型看见了 3D，所以动作更准”，而是“模型先形成一个可操控的 3D 未来假设，再由此出动作”。LaMP、PointWorld、DreamZero、Ctrl-World 其实都在往这个方向靠，只是载体不一样：一个是 scene flow，一个是 point flow，一个是 video latent，一个是 imagination rollouts。这个趋势我觉得会比“再做一个更强的 3D VLA encoder”更长久。

主线最好定成“3D 运动表征如何成为动作决策的中间层”，而不是泛泛地做“VLA+3D”。

DA3 ​

核心

用一个尽量简单的 Transformer，统一处理单目深度、多视图几何、相机位姿估计，甚至进一步接到 3D Gaussian 表示上。从任意数量视角恢复 visual space”，而且输入图像可以有相机位姿，也可以没有。

它在方法论上和 VGGT 那类“多头、多任务、专门几何结构”的路线有点不一样。DA3 主张一个 plain transformer，不一定需要很复杂的专门几何架构；同时，它不想再显式预测一大堆互相关联的几何量，而是用一个统一的 depth-ray representation 来做建模。

给你 1 张、2 张、很多张图，甚至一段视频，模型能不能都用同一套骨干网络来理解“这个空间长什么样”。DA3 给出的答案是可以。同一个 any-view 主系列模型，只要换输入配置，就能做单目深度估计、多视图一致深度、pose-conditioned depth、相机位姿估计，甚至还能做 3D Gaussian estimation。这和过去“单目模型一套、多视图模型一套、位姿估计再一套”的分裂式设计很不一样。

输出：强调空间一致性，它会输出一致的 depth 和 ray maps，从而在不同视角之间形成可以融合的几何结构；DA3 不只是“做深度”，而是在做一个更完整的 visual geometry foundation model。Depth Anything 2 更偏强单目深度，而 DA3 把任务扩展到了 any-view geometry。DA3 不仅在多视图深度和相机位姿上超过了 VGGT，而且在单目深度上也超过了 DA2。它想用一套统一模型把“从单目到多视图”的几何能力都包进去。

性能：DA3 在它们新建的 visual geometry benchmark 上，相比此前 SOTA 的 VGGT，平均提升了 35.7% 的相机位姿精度 和 23.6% 的几何精度；arXiv 摘要里给出的数字更高，写的是 44.3% pose accuracy 和 25.1% geometric accuracy。这两个数字不完全一致，通常可以理解为项目页与论文版本在评测汇总或版本更新上有差异，但共同结论是一致的：DA3 在发布时把 VGGT 压过去了。

延伸点：第一，官方明确提供了 DA3-Streaming，说可以用滑窗式 streaming inference 处理超长视频，并把显存压到 12GB 以下。这说明 DA3 不只是静态多视图模型，也在往长序列视频几何建模方向扩。第二，它原生支持 3D Gaussian estimation，官方 repo 和模型卡都把这一项列成主能力；项目页还专门展示了“冻结 backbone、再训练一个 DPT head 来预测 3DGS 参数”的 feed-forward 3D Gaussians 结果。

对比：VGGT 更像“统一预测 camera / depth / point map / tracking feature 的几何大模型”，而 DA3 更强调“统一的 depth-ray 目标 + plain transformer 足以把这些空间关系学出来”。前者更像通过多种显式几何输出联合约束空间结构，后者更像在表示层面做了极简化，试图用更少的任务头、更统一的目标去覆盖更广的视角输入。DA3 的方法哲学比 VGGT 更“minimalist”。

应用：2026 年的一篇工作直接把 DA3 的 feature space 拿来当扩散模型的 latent space，以引入跨视角几何先验；还有一篇做任务感知观测接口的机器人论文，把 Depth Anything 3 用作几何通道，为策略提供更稳定的语义-几何观测；另一篇最近的测试时自监督校正工作也 把 DA3 和 VGGT 都作为可提升的基础几何模型。

DA3 是把 Depth Anything 从“强单目深度模型”推进成“任意视角 3D 几何基础模型”的版本。它提出：3D 几何大模型不一定要靠复杂专用结构堆出来，一个 plain transformer 加统一 depth-ray 表示，也能把单目、多视图、位姿和 3DGS 这几件事较优雅地统一起来。
